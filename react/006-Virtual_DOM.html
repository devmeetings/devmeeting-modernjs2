Najważniejszą częścią Reacta jest Virtual DOM

Założenia Reacta:

  • Operacje na drzewie DOM są kosztowne (w sensie czasu),
  • a minimalizowanie tych operacji jest trudne do implementacji.

Skąd React wie, które części DOM należy re-renderować?

Funkcja render nie zwraca gotowego HTML/DOM do wstawienia - zamiast tego zwracany jest VirtualDOM.

VirtualDOM to lekka reprezentacja tego co chcemy, żeby znalazło się w drzewie DOM. React porównuje nowo zwróconą wartość z poprzednią i aplikuje tylko minimalną liczbę wymaganych zmian.
